## Overview

Optimize InfraGuard for handling thousands of metrics and high-volume alerting.

## Key Performance Metrics

| Metric | Target | Description |
|--------|--------|-------------|
| Collection Latency | < 5s | Time to collect all metrics |
| Detection Latency | < 1s | Time to detect anomalies |
| API Response Time | < 200ms | API endpoint response time |
| Memory Usage | < 2GB | Container memory footprint |
| CPU Usage | < 50% | Average CPU utilization |

## Configuration Tuning

### API Server

```yaml
api:
  workers: 8              # 2x CPU cores
  timeout: 60s
  max_connections: 1000
  
  rate_limiting:
    enabled: true
    requests_per_minute: 1000
    burst: 100
```

### Collector

```yaml
collector:
  prometheus:
    scrape_interval: 60s  # Balance freshness vs load
    timeout: 30s
    max_retries: 3
    batch_size: 100       # Metrics per batch
  
  performance:
    parallel_queries: 4   # Concurrent Prometheus queries
    cache_results: true
    cache_ttl: 30s
```

### Detector

```yaml
detector:
  isolation_forest:
    n_estimators: 100     # More = better accuracy, slower
    max_samples: 256      # Lower = faster training
  
  training:
    parallel_models: 4    # Train multiple metrics concurrently
    batch_size: 1000
  
  performance:
    cache_predictions: true
    cache_ttl: 60s
```

### Database

```yaml
database:
  type: "postgresql"      # Use PostgreSQL for production
  
  postgresql:
    pool_size: 20
    max_overflow: 40
    pool_timeout: 30s
    pool_recycle: 3600
```

## Caching Strategy

### Memory Cache

```yaml
performance:
  caching:
    enabled: true
    backend: "memory"
    ttl: 300s
    max_size_mb: 512
```

### Redis Cache

```yaml
performance:
  caching:
    enabled: true
    backend: "redis"
    ttl: 300s
  
  redis:
    host: "redis"
    port: 6379
    db: 0
    max_connections: 50
```

## Resource Limits

### Docker

```yaml
services:
  infraguard:
    deploy:
      resources:
        limits:
          cpus: '4'
          memory: 4G
        reservations:
```

## Monitoring Performance

```bash
# API latency
curl http://localhost:8000/metrics | grep api_request_duration

# Collection performance
curl http://localhost:8000/metrics | grep collector_duration

# Detection performance
curl http://localhost:8000/metrics | grep detector_duration

# Resource usage
docker stats infraguard
```

## Scaling Strategies

### Vertical Scaling

Increase resources for single instance:
- 4-8 CPU cores
- 8-16GB RAM
- SSD storage
- Tune worker counts

### Horizontal Scaling

Deploy multiple instances:
- Load balancer in front
- Shared PostgreSQL database
- Redis for distributed caching
- Metric sharding by host/service
