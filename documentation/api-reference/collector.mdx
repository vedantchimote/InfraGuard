## Overview

The Collector API manages metric collection from Prometheus and other data sources.

## Endpoints

### Get Metrics

```http
GET /api/collector/metrics
```

Retrieve collected metrics with optional filtering.

<ParamField query="metric_name" type="string">
  Filter by metric name
</ParamField>

<ParamField query="start_time" type="string">
  Start time (ISO 8601)
</ParamField>

<ParamField query="end_time" type="string">
  End time (ISO 8601)
</ParamField>

<ResponseExample>
```json
{
  "metrics": [
    {
      "name": "cpu_usage_percent",
      "timestamp": "2026-04-06T10:30:00Z",
      "value": 75.5,
      "labels": {
        "instance": "prod-web-01",
        "job": "node-exporter"
      }
    }
  ],
  "count": 1
}
```
</ResponseExample>

### Trigger Collection

```http
POST /api/collector/collect
```

Manually trigger metric collection.

<ResponseExample>
```json
{
  "status": "success",
  "metrics_collected": 42,
  "duration_ms": 1250
}
```
</ResponseExample>

### Add Custom Metric

```http
POST /api/collector/metrics
```

Add a custom metric definition.

<ParamField body="name" type="string" required>
  Metric name
</ParamField>

<ParamField body="query" type="string" required>
  PromQL query
</ParamField>

<ParamField body="labels" type="array">
  Label names to extract
</ParamField>

<ResponseExample>
```json
{
  "status": "success",
  "metric_id": "metric_123"
}
```
</ResponseExample>

## Python SDK

```python
from infraguard import Collector

collector = Collector(config)

# Get metrics
metrics = collector.get_metrics(
    metric_name="cpu_usage",
    start_time="2026-04-06T00:00:00Z",
    end_time="2026-04-06T23:59:59Z"
)

# Trigger collection
result = collector.collect()

# Add custom metric
collector.add_metric(
    name="custom_metric",
    query="rate(http_requests_total[5m])",
    labels=["instance", "status"]
)
```
